# Multi-Site Web Portfolio Unification

**Role:** Webmaster / Frontend Developer  
**Stack:** WordPress · Google Sites · Green Digital Design · Core Web Vitals · Confluence  
**Type:** Web Operations · Portfolio Consolidation · Performance Optimization

---

## Overview

Consolidated three separate campus web properties into a single, unified, standards-aligned digital portfolio. Reduced operational overhead, eliminated redundant content, and improved performance scores across all Core Web Vitals metrics.

## Problem

Three independently maintained campus web properties had grown out of sync:
- A WordPress site, a Google Sites property, and a legacy static HTML site
- Duplicated content across all three with no single source of truth
- Heavy page assets causing poor LCP and CLS scores
- No consistent design language or navigation structure

## What I Did

- Audited all three sites for content overlap, asset weight, and structural issues
- Developed a unified information architecture consolidating all content into one standards-aligned portfolio
- Applied green digital design principles: removed redundant assets, optimized image weight, reduced page bloat
- Improved Core Web Vitals — Largest Contentful Paint (LCP) and Cumulative Layout Shift (CLS)
- Maintained a Confluence backlog throughout the project, distributing release notes each sprint cycle
- Coordinated with stakeholders across departments to ensure no critical content was lost in migration

## Outcome

- Three sites reduced to one unified, maintainable portfolio
- Measurable improvement in Core Web Vitals scores
- Reduced page weight through image optimization and asset removal
- Streamlined content governance with a single update workflow

## Key Principles Applied

- Green digital design (reduced carbon footprint of web assets)
- Core Web Vitals optimization (LCP, CLS, FID)
- Agile delivery with Confluence documentation
- Mobile-first responsive design
